# Changelog

## v0.8.9 - 2026-05-07

### Added
- 为 `IR` 万能遥控新增 5 个独立红外库入口，将列表从 9 项扩展为 14 项。
- 新增 `电视补充`、`投影补充`、`音响补充`、`广场音响`、`风扇补充` 五个预置库选项。

### Changed
- 将补充红外库移动到 `applications/main/infrared/resources/infrared/assets/` 下独立管理，不再并入原有分类库文件。
- 调整 `IR` 万能遥控的预置库载入逻辑，使新增库可直接作为内置列表项打开，无需额外文件浏览步骤。

## v0.8.8 - 2026-05-06

### Changed
- 回退了 `v0.8.7` 中的回退操作，恢复了此前被撤销的修改。
- 调整并改进了 `Sub-GHz` 相关内容。

### Fixed
- 修复 `IR` 万能遥控因按钮名称本地化导致的闪退重启问题。

## v0.8.7 - 2026-05-06

### Changed
- 回退了 `v0.8.6` 和 `v0.8.5` 中的相关修改。
- 调整并优化 `mlib`。

## v0.8.6 - 2026-05-06

### Changed
- 更新 `FreeRTOS-Kernel` 到 `V10.6.2`。

### Improved
- 进一步补齐 `10.6.1` / `10.6.2` 对内核参数检查的强化，减少异常参数落入 `assert` 之前的未定义行为风险。
- 同步 `10.6.2` 对新 MPU wrapper 的整数溢出修复、运行时参数校验与系统调用入口改进，为后续内核安全边界增强打下基础。

### Fixed
- 保持 `v0.8.5` 已获得的容量改善，不引入额外的主固件体积回退。

## v0.8.5 - 2026-05-06

### Changed
- 更新 `FreeRTOS-Kernel` 到 `V10.6.0`。

### Improved
- 改进 FreeRTOS 调度器在“同优先级任务变为就绪”时的处理逻辑，减少不必要的抢占，降低事件循环、定时器与后台任务之间的切换抖动。
- 同步 `heap_4` / `heap_5` 的对齐填充修正，减少不必要的堆内存浪费，对当前固件的 RAM 使用更友好。
- 同步 `uxTaskGetSystemState` / `eTaskGetState` 的状态返回修正，提升线程状态观察与运行时诊断的一致性。
- 引入 `ulTaskGetRunTimeCounter` / `ulTaskGetRunTimePercent` 等运行时统计能力增强，为线程与性能诊断提供更稳定的底层支持。
- 同步内核对象静态 buffer 获取接口与系统调用路径改进，为后续内核资源管理和安全边界增强打下基础。

### Fixed
- 缓解主固件打包阶段的容量压力，降低 `updater_package` 因接近 C2 区域而失败的风险。

## v0.8.4 - 2026-05-06

### Changed
- 同步 `stm32wbxx_hal_driver` 至 `v1.14.7`，引入最新 STM32WB 外设驱动修复与兼容性改进。
- 同步 `nanopb` 至 `0.4.9.1`，吸收上游稳定性与解析安全性修复。
- 同步 `libusb_stm32` 至 `b9d70a0`，补齐 USB MSC Bulk 相关支持并改进 USB 传输兼容性。
- 同步 `cmsis-device-wb` 至 `v1.12.3`，更新 STM32WB 设备头文件、启动文件及底层定义。
- 同步 `doxygen-awesome-css` 至 `v2.4.2`，更新文档主题与样式细节。

### Benefits
- 提升底层 STM32WB 驱动栈的一致性，降低外设边界问题和隐藏兼容性风险。
- 改善 USB 相关功能的稳定性，对大容量传输和 MSC 场景更友好。
- 提高 `nanopb` 数据解析链路的可靠性，减少序列化与反序列化异常风险。
- 对齐 CMSIS 与 HAL 版本，减少头文件、寄存器定义和启动代码不一致带来的维护成本。
- 让后续同步上游补丁与排查底层问题更直接，降低长期维护负担。
- 优化文档样式显示，提升项目文档的可读性与一致性。
