# 汉化方法

## 历史方案

### `CN_built-in`

`CN_built-in` 的思路是：

- 直接修改源码里的中文文本
- 使用内置字库文件
- 通过 `canvas_set_custom_u8g2_font()` 或类似方式直接切到内置中文字库

这个方案的特点是实现直接，但问题也明显：

- 字库和代码耦合重
- 后续同步上游时冲突多
- 每次增减汉字都要重新维护内置字体源

### `CN`

`CN` 在 `CN_built-in` 基础上继续演进，思路变成：

- 仍然直接修改源码里的中文文本
- 字体不再完全内置，转向资源包方式
- 通过 `asset_packs` 下的字体资源完成中文显示

这个方案比 `CN_built-in` 灵活，但也有问题：

- 中文字体依赖 `asset_packs`
- 需要切换资源包
- 中文主字库和主题资源包系统耦合
- 运行时路径和维护边界不够清晰

## 当前方案

当前 `CN-Clipper` 不再沿用上面两种方法。

现在的方案是：

1. 编译时收集中文词条、源码中文字符串、动画文本和 `application.fam` 的 `name_zh`
2. 生成总字库 `primary_zh.c` 和 `primary_zh.u8f`
3. 主固件把 `primary_zh.c` 链接进固件资源库
4. 运行时由 `canvas` 检测非 ASCII 文本，并临时切换到内置 `primary_zh` 绘制
5. RAM updater 阶段不链接完整中文字库，避免更新器超过体积限制

默认英文固件不启用这套中文路径。

## 当前实现

### 编译时

中文构建时会读取：

- `localization/zh_CN/strings.json`
- 源码里的中文字符串
- `assets/dolphin/**/meta.txt` 中的中文动画文本
- `applications/**/application.fam` 中的 `name_zh`

并通过：

- `scripts/momentum_zh_font_gen.py`
- `tools/u8g2_cn_tools/bdfconv`
- `tools/u8g2_cn_tools/bdf/fusion-pixel-10px-proportional-zh_hans.bdf`

生成：

- `build/<target>/generated/zh_fonts_work/primary_zh.map`
- `build/<target>/generated/zh_fonts_work/primary_zh.c`
- `build/<target>/generated/zh_fonts/primary_zh.u8f`

主固件实际使用的是：

- `build/<target>/generated/zh_fonts_work/primary_zh.c`

`primary_zh.u8f` 仍会作为资源输出，可用于手动替换或调试字体文件。

### 运行时

`canvas` 在检测到字符串包含非 ASCII 字节时，会尝试使用：

- `primary_zh`

然后临时切换字体，再调用 `u8g2_DrawUTF8()` 完成绘制。

这意味着：

- 不再依赖 `asset_packs/CN-Clipper/Fonts`
- 不再要求手动切换资源包
- 主固件显示中文不依赖 SD 卡字体文件
- updater 阶段会回退到英文/内置字体路径

## 构建方式

### 英文固件

```bash
./fbt COMPACT=1 DEBUG=0
```

或：

```bash
./fbt updater_package COMPACT=1 DEBUG=0
```

### 中文固件

```bash
MOMENTUM_UI_LANG=zh_CN ./fbt COMPACT=1 DEBUG=0
```

或：

```bash
MOMENTUM_UI_LANG=zh_CN ./fbt updater_package COMPACT=1 DEBUG=0
```

只有在 `MOMENTUM_UI_LANG=zh_CN` 时，才会生成并打包 `primary_zh.u8f`。

### 生成全部可选字体

```bash
./fbt zh_fonts_all
```

这个目标会把 `tools/u8g2_cn_tools/bdf/` 下的 BDF 字体都转换为 `.u8f`，用于 Release 字体包或手动替换测试。

## 词条维护方式

当前中文字符来源包含源码扫描，但核心 UI 文案应优先维护在：

- `localization/zh_CN/strings.json`

新增核心 UI 中文时，应优先更新这里，再触发编译生成新字库。

## 当前目录说明

当前正确的中文字体资源目录是：

- `zh_fonts/primary_zh.u8f`

注意：这是资源输出目录，不是主固件运行时加载中文字体的唯一来源。当前主固件优先使用已链接进固件的 `primary_zh`。
